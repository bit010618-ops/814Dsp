# Codex 工作规则

## 仓库与 Git 边界

- 固定仓库：`bit010618-ops/814Dsp`；先核对 `origin`，默认保持当前分支不变。
- 保留一切既有未提交或未跟踪内容；不得 reset、clean、stash、强制覆盖或隐式暂存。
- 只暂存当前任务明确允许的文件，逐个列出路径；禁止 `git add .` 和 `git add -A`。
- 发现 merge/rebase 冲突时停止相关写入并报告；push 失败时不 pull、rebase 或 force-push。

## 最小恢复流程

每次新会话、上下文压缩或恢复后，默认只读取：

1. `AGENTS.md`
2. `.codex/CONTEXT.md`
3. `.codex/TASK.md`
4. `git status --short`、`git log -3 --oneline`、`git diff --name-only`

随后仅按 `.codex/TASK.md` 所列路径按需读取和执行。不要默认全文读取 `HANDOFF.md`、两个 requirements 文件，也不要默认检查 `sample/`、`outputs/` 或 `full/` 全目录。

## 读取与 Token 控制

- 优先使用 `rg`、`git grep` 和精确路径；搜索结果过长时缩小范围，只读必要行附近内容。
- 不全仓扫描、不重读长历史、不在回复中粘贴完整大文件或长日志。
- `HANDOFF.md` 仅为兼容入口；详细历史在其归档中，只有任务确实需要时再读。

- 原课件的唯一信息不得遗漏，章节结构、知识点与顺序沿用原课件。
- 成品为 A4 纵版、彩色、传统教材风，适合打印、装订、学习与考研复习。
- 所有公式必须正式数学排版；上下标、根号、求和及符号完整，分式必须竖式。
- 清理课件模板或水印不得损失图中的坐标、曲线、公式、单位、标签或其他有效信息；必要时修复或重绘。
